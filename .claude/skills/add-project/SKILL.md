Add a static project page to this dnuke.com site.

Synced GitHub projects get their pages auto-generated by `sync-projects.sh` and committed to `src/projects/<slug>/index.md`. Static projects (art installations, offline work, anything without a repo) are written directly in the same format.

## What to collect from the user

If $ARGUMENTS is provided, use it as the slug (lowercase, hyphens, no spaces). Otherwise ask.

Collect the following — ask for anything not already provided in the conversation:
- **title** — display name (e.g. "Diamond Seas Installation")
- **slug** — URL-safe identifier (e.g. `diamond-seas`)
- **description** — one sentence for the project card on the home page
- **tags** — comma-separated list (e.g. `installation, sculpture, light, video`)
- **github** — GitHub URL if one exists, otherwise leave empty
- **has_demo** — true if there's a live demo at `/<slug>/`, otherwise false

## Files to create

**`src/projects/<slug>/index.md`** — the project landing page:

```markdown
---
layout: project-page.njk
title: "<title>"
slug: <slug>
description: "<description>"
tags: [<tags as yaml list>]
github: "<github url or empty string>"
has_demo: false
permalink: /projects/<slug>/
---

<body content from the conversation — photos, description, context>
```

**`src/images/projects/<slug>/`** — create this directory with a `.gitkeep` so it's ready for photos:
```
touch src/images/projects/<slug>/.gitkeep
```

## Notes

- Photos go in `src/images/projects/<slug>/` and reference as `![alt](/images/projects/<slug>/photo.jpg)`
- The `src/images/` directory is passthrough-copied by Eleventy so images are served as-is
- The project will appear on the home page automatically via `collections.projects`
- If the project has no devlog, the Devlog link on the card will 404 — that's acceptable, or omit by not having `slug` match any devlog entries
- Do NOT add this project to `projects.toml` — that file is only for synced GitHub repos
